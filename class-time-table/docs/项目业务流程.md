# 电子班牌项目
---
这是一个内网下的全双工通信项目  主要用于iot设备信息展示、门禁控制
这里的技术栈主要是java(server) kotlin(android client)
以下是实际业务的几个流程

---
1. 班牌设备上电后向服务端发送ws请求handshake，此时班牌未有任何配置，仅有default.json告知的服务器地址,当handshake成功后，触发onAfterHandShake回调，由服务端下发班牌的默认配置信息，这里视作握手完成，客户端应用服务端下发的配置文件来注册全局Config单例，供后续compose组件调用，同时server端通过JimServerApi.bindClassTimeTable()完成通道绑定，并通过AbstractImClassTimeTableListener实现后置回调，触发班牌信息持久化到redis
2. server端有spring web servlet http服务供管理员使用webui配置对应班牌的config，当管理员配置完成config时，找到对应channel发送packet告知客户端刷新全局Config单例(这里必须synchronized 锁死object 防止错误配置被compose读取导致的exception), 同时要等待client响应拿到resp 回复CompleteFuture告知用户真正完成了配置，并重新调用JimServerApi.bindClassTimeTable 调用链完成持久化等功能
3. 管理员使用上述webui对班牌下发packet包装有bitmap，班牌提取packet中的bitmap尝试录入班牌侧本地人脸识别库，本地人脸识别有验证bitmap是否合规的功能，若合规并成功录入后，告知server侧completefuture从而通知管理员人脸照片插入成功，和流程2一致，操作超时时默认返回失败
