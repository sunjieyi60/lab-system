# 设备注册流程说明（RSA公钥加密方案）

## 1. 方案概述

### 1.1 设计思路

采用**RSA公钥加密**方案：
- **设备持有服务端公钥**：设备使用服务端RSA公钥加密设备信息
- **服务端私钥解密**：服务端用私钥解密，如果解密成功则设备合法
- **ChannelContext绑定**：注册成功后绑定ChannelContext的bindId为设备ID
- **Redis验证**：后续请求通过ChannelContext的bindId在Redis中验证设备合法性
- **无需Token**：利用t-io框架的ChannelContext绑定机制，无需额外的token机制

### 1.2 优势

1. **简单高效**：直接RSA加密，无需混合加密
2. **无需预注册**：设备信息通过加密传输，服务端解密后自动注册
3. **天然身份验证**：利用t-io的ChannelContext绑定机制，无需token
4. **安全性高**：只有持有服务端公钥的设备才能成功注册
5. **自动刷盘**：Redis临时存储，定时任务批量刷盘MySQL

## 2. 注册请求格式

### 2.1 REGISTER指令Payload格式

**重要**：packet的payload直接是**RegisterRequestDTO序列化后的JSON字符串经公钥加密的byte[]**

**RegisterRequestDTO（加密前）示例：**
```json
{
    "deviceInfo": {
        "deviceId": "BOARD_101",
        "deviceName": "101教室班牌",
        "macAddress": "AA:BB:CC:DD:EE:FF",
        "version": "1.0.0",
        "deviceType": "smartboard",
        "hardwareInfo": "{\"cpu\":\"RK3568\",\"ram\":\"4GB\"}"
    }
}
```

**加密流程：**
1. 构建RegisterRequestDTO对象（包含DeviceInfoDTO）
2. 序列化为JSON字符串
3. 使用服务端RSA公钥加密JSON字符串（如果超过245字节需要分片）
4. 得到加密后的byte[]，**直接作为packet的payload**

## 3. 注册响应格式

### 3.1 REGISTER_ACK指令Payload格式

**重要**：服务端回复时使用**私钥签名payload**，保证信息完整性和身份验证

**成功响应（签名前）：**
```json
{
    "success": true,
    "token": null,
    "config": {
        "heartbeatInterval": 5000,
        "reconnectDelay": 5000,
        "chunkSize": 524288,
        "features": {
            "faceRecognition": true,
            "accessControl": true,
            "timetableSync": true,
            "otaUpdate": true
        },
        "server": {
            "host": "192.168.10.100",
            "port": 9999
        }
    }
}
```

**签名流程：**
1. 将RegisterAckDTO序列化为JSON字符串
2. 使用服务端私钥签名JSON字符串（SHA256withRSA）
3. 将签名Base64编码
4. payload格式：`{JSON字符串}|{签名Base64}`

**失败响应（签名前）：**
```json
{
    "success": false,
    "errorCode": "DECRYPT_FAILED",
    "errorMessage": "设备信息解密失败，设备不合法"
}
```

**注意**：
- `token`字段为`null`，因为使用ChannelContext的bindId进行身份验证，无需token
- 客户端收到响应后，需要验证签名，确保响应来自合法的服务端

**错误代码列表：**
- `DECRYPT_FAILED`: 设备信息解密失败（设备不合法）
- `INVALID_DEVICE`: 设备ID为空
- `INVALID_FORMAT`: 请求格式错误
- `INTERNAL_ERROR`: 服务器内部错误

## 4. 注册流程

### 4.1 客户端流程

1. **准备设备信息**
   ```java
   DeviceInfoDTO deviceInfo = DeviceInfoDTO.builder()
       .deviceId("BOARD_101")
       .deviceName("101教室班牌")
       .macAddress("AA:BB:CC:DD:EE:FF")
       .version("1.0.0")
       .deviceType("smartboard")
       .hardwareInfo("{\"cpu\":\"RK3568\"}")
       .build();
   ```

2. **加密注册请求**
   ```java
   String serverPublicKey = loadServerPublicKey(); // 从配置或文件加载服务端公钥
   // RegisterRequestDTO序列化后加密，直接得到byte[]
   byte[] encryptedPayload = DeviceEncryptionUtil.encryptRegisterRequest(
       deviceInfo, 
       serverPublicKey
   );
   ```

3. **发送REGISTER请求**
   ```java
   // payload直接是加密后的byte[]
   SmartBoardPacket packet = packetBuilder.build(
       CommandType.REGISTER, 
       encryptedPayload,  // 直接使用加密后的byte[]
       QosLevel.AT_LEAST_ONCE
   );
   Tio.send(channelContext, packet);
   ```

4. **接收并验证REGISTER_ACK响应**
   ```java
   // 接收响应payload
   byte[] responsePayload = ackPacket.getPayload();
   String responseStr = new String(responsePayload, StandardCharsets.UTF_8);
   
   // 解析payload：{JSON字符串}|{签名Base64}
   String[] parts = responseStr.split("\\|");
   if (parts.length != 2) {
       log.error("响应格式错误");
       return;
   }
   
   String responseJson = parts[0];
   String signatureBase64 = parts[1];
   
   // 验证签名（确保响应来自合法的服务端）
   boolean verified = DeviceEncryptionUtil.verifySignature(
       responseJson, 
       signatureBase64, 
       serverPublicKey
   );
   
   if (!verified) {
       log.error("响应签名验证失败");
       return;
   }
   
   // 解析响应
   RegisterAckDTO ackDTO = JSON.parseObject(responseJson, RegisterAckDTO.class);
   if (ackDTO.getSuccess()) {
       DeviceConfigDTO config = ackDTO.getConfig();
       // 保存配置，开始正常通信
       // 注意：无需保存token，ChannelContext已自动绑定设备ID
   }
   ```

### 4.2 服务端流程

1. **接收REGISTER请求**
2. **解密payload**：
   - packet的payload直接是加密后的byte[]
   - 使用服务端RSA私钥解密byte[]
   - 如果解密失败，说明设备不合法，**中断连接**
   - 解密后得到RegisterRequestDTO的JSON字符串
   - 反序列化为RegisterRequestDTO，提取DeviceInfoDTO
3. **写入Redis**：将设备信息写入Redis（key: `smartboard:device:register:{deviceId}`）
4. **绑定ChannelContext**：调用`Tio.bindBsId(channelContext, deviceId)`，将设备ID绑定到ChannelContext
5. **获取设备配置**：根据设备类型获取配置（或使用默认配置）
6. **签名响应**：
   - 构建RegisterAckDTO并序列化为JSON字符串
   - 使用服务端私钥签名JSON字符串（SHA256withRSA）
   - 将签名Base64编码
   - payload格式：`{JSON字符串}|{签名Base64}`
7. **发送REGISTER_ACK**：发送签名后的payload

### 4.3 后续请求验证流程

1. **接收请求**：Handler收到非REGISTER指令
2. **验证设备合法性**：
   - 获取ChannelContext的bindId（即设备ID）
   - 检查Redis中是否存在该设备（key: `smartboard:device:register:{deviceId}`）
   - 如果不存在，说明设备未注册，**关闭连接**
3. **处理请求**：设备合法，继续处理业务逻辑

### 4.4 Redis到MySQL刷盘流程

1. **定时任务**：每5分钟执行一次（`@Scheduled(fixedRate = 300000)`）
2. **扫描Redis**：查找所有 `smartboard:device:register:*` 的key
3. **读取设备信息**：从Redis读取设备信息
4. **检查MySQL**：检查设备是否已存在
5. **插入或更新**：
   - 新设备：插入到MySQL
   - 已存在：更新设备信息（保留创建时间）
6. **保留Redis数据**：刷盘后保留Redis数据（用于设备验证），等待过期（24小时）

## 5. 密钥管理

### 5.1 生成服务端密钥对

```bash
# 生成RSA密钥对（2048位）
openssl genrsa -out server_private_key.pem 2048
openssl rsa -in server_private_key.pem -pubout -out server_public_key.pem

# 将私钥转换为PKCS8格式（Java需要）
openssl pkcs8 -topk8 -inform PEM -in server_private_key.pem -outform PEM -nocrypt -out server_private_key_pkcs8.pem

# 将公钥转换为Base64（用于客户端配置）
cat server_public_key.pem | base64 -w 0
```

### 5.2 服务端配置

```yaml
device:
  auth:
    server:
      # 方式1：直接配置Base64编码的私钥
      private-key: "MIIEvQIBADANBgkqhkiG9w0BAQEFAASCBKcwggSjAgEAAoIBAQC..."
      
      # 方式2：从文件读取私钥（推荐，更安全）
      private-key-file: "classpath:keys/server_private_key_pkcs8.pem"
```

### 5.3 客户端配置

```properties
# 服务端公钥（Base64编码或PEM文件路径）
server.public.key=MIIBIjANBgkqhkiG9w0BAQEFAAOCAQ8AMIIBCgKCAQEA...
# 或
server.public.key.file=/path/to/server_public_key.pem
```

## 6. 数据库设计

### 6.1 设备表结构

```sql
CREATE TABLE `smart_board_device` (
    `device_id` VARCHAR(64) NOT NULL COMMENT '设备ID（唯一标识）',
    `device_name` VARCHAR(128) DEFAULT NULL COMMENT '设备名称',
    `mac_address` VARCHAR(32) DEFAULT NULL COMMENT '设备MAC地址',
    `ip_address` VARCHAR(64) DEFAULT NULL COMMENT '设备IP地址',
    `device_type` VARCHAR(32) DEFAULT NULL COMMENT '设备类型',
    `hardware_info` TEXT DEFAULT NULL COMMENT '硬件信息JSON',
    `status` VARCHAR(16) DEFAULT 'OFFLINE' COMMENT '设备状态：ONLINE, OFFLINE',
    `version` VARCHAR(32) DEFAULT NULL COMMENT '设备版本号',
    `last_online_time` DATETIME DEFAULT NULL COMMENT '最后在线时间',
    `create_time` DATETIME DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
    `update_time` DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
    `config_json` TEXT DEFAULT NULL COMMENT '设备配置JSON',
    PRIMARY KEY (`device_id`),
    INDEX `idx_mac` (`mac_address`),
    INDEX `idx_status` (`status`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='智能班牌设备表';
```

**注意**：此方案中不需要 `public_key` 字段，因为设备信息通过加密传输，服务端解密后自动注册。

## 8. 安全说明

1. **RSA公钥加密**：设备使用服务端公钥加密RegisterRequestDTO，只有服务端私钥能解密
   - 保证client的身份合法性：只有持有服务端公钥的设备才能成功注册
   - 保证信息的秘密性：设备信息加密传输，只有服务端能解密
2. **RSA私钥签名**：服务端使用私钥签名响应payload
   - 保证响应完整性：防止响应被篡改
   - 保证服务端身份：客户端验证签名确保响应来自合法服务端
3. **设备合法性验证**：解密成功即设备合法，解密失败立即中断连接
4. **ChannelContext绑定**：注册成功后绑定设备ID，后续通过bindId验证
5. **Redis验证**：每次请求检查ChannelContext的bindId是否在Redis中
6. **自动注册**：无需预注册，设备信息通过加密传输自动注册上云

### 9.1 注册时绑定ChannelContext

```java
// 绑定ChannelContext的bindId为设备ID
Tio.bindBsId(channelContext, deviceId);
```

### 9.2 后续请求验证

```java
// 获取ChannelContext的bindId（即设备ID）
String bindId = channelContext.getBsId();

// 检查Redis中是否存在该设备
String registerKey = "smartboard:device:register:" + bindId;

}
```

## 10. 客户端加密工具使用示例

```java
// 1. 准备设备信息
DeviceInfoDTO deviceInfo = DeviceInfoDTO.builder()
    .deviceId("BOARD_101")
    .deviceName("101教室班牌")
    .macAddress("AA:BB:CC:DD:EE:FF")
    .version("1.0.0")
    .deviceType("smartboard")
    .hardwareInfo("{\"cpu\":\"RK3568\"}")
    .build();

// 2. 加密设备信息（需要服务端公钥）
String serverPublicKey = "MIIBIjANBgkqhkiG9w0BAQEFAAOCAQ8AMIIBCgKCAQEA...";
String encryptedPayload = DeviceEncryptionUtil.encryptDeviceInfo(deviceInfo, serverPublicKey);

// 3. 构建注册请求
RegisterRequestDTO request = new RegisterRequestDTO();
request.setEncryptedPayload(encryptedPayload);

// 4. 序列化并发送
String json = JSON.toJSONString(request);
byte[] payload = json.getBytes(StandardCharsets.UTF_8);
SmartBoardPacket packet = packetBuilder.build(CommandType.REGISTER, payload, QosLevel.AT_LEAST_ONCE);
Tio.send(channelContext, packet);
```

## 11. RSA加密长度限制说明

RSA-2048密钥最多加密245字节数据。如果设备信息JSON超过245字节，需要分片加密：

1. **客户端**：将JSON分片（每245字节一片），分别加密后拼接
2. **服务端**：将加密数据分片（每256字节一片），分别解密后拼接

代码已自动处理分片逻辑，无需手动处理。
