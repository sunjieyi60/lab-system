# 父子组件参数回传说明

## 背景与目标
这里的“子组件回传参数给父组件”指的是：父组件传入回调函数给子组件，子组件在需要时调用回调，把数据或事件上行给父组件。父组件收到回调后更新自身状态，从而触发 UI 更新。

当前项目中对应的实现主要集中在 DoorOpeningDialog 与 PwdOpenUiProvider：
- 父组件：DoorOpeningDialog 持有 timeoutMillis，并把 onClose 与 onTimeoutUpdate 传给子组件
- 子组件：PwdOpenUiProvider 在倒计时更新时调用 onTimeoutUpdate，在结束或用户操作时调用 onClose

## 传递流程（简要）
1. 父组件持有状态（如 timeoutMillis）
2. 父组件把回调函数传给子组件
3. 子组件在业务逻辑中调用回调上行数据
4. 父组件在回调中更新状态，触发 Compose 重组

## Kotlin 简化 Demo（Compose）
下面演示父组件通过回调接收子组件的倒计时更新，并关闭弹窗：

```kotlin
import androidx.compose.runtime.*
import androidx.compose.material3.*
import kotlinx.coroutines.delay

@Composable
fun ParentDialog() {
    var visible by remember { mutableStateOf(true) }
    var remaining by remember { mutableStateOf(5000L) }
    if (visible) {
        ChildPanel(
            onTimeoutUpdate = { remaining = it },
            onClose = { visible = false }
        )
    }
    Text(text = "${remaining / 1000}s")
}

@Composable
fun ChildPanel(
    onTimeoutUpdate: (Long) -> Unit,
    onClose: () -> Unit
) {
    var left by remember { mutableStateOf(5000L) }
    LaunchedEffect(Unit) {
        while (left > 0L) {
            delay(1000)
            left -= 1000
            onTimeoutUpdate(left)
        }
        onClose()
    }
    Text(text = "$left")
}
```

## Java 简化 Demo（回调上行）
下面演示子组件在循环中回传剩余时间给父组件：

```java
import java.util.function.LongConsumer;

public class ParentDemo {
    private long remaining = 5000L;
    private boolean visible = true;

    public static void main(String[] args) throws Exception {
        ParentDemo parent = new ParentDemo();
        ChildDemo child = new ChildDemo(
            parent.remaining,
            v -> { parent.remaining = v; System.out.println("remaining=" + v); },
            () -> { parent.visible = false; System.out.println("visible=" + parent.visible); }
        );
        child.run();
    }
}

class ChildDemo {
    private long left;
    private final LongConsumer onTimeoutUpdate;
    private final Runnable onClose;

    public ChildDemo(long initial, LongConsumer onTimeoutUpdate, Runnable onClose) {
        this.left = initial;
        this.onTimeoutUpdate = onTimeoutUpdate;
        this.onClose = onClose;
    }

    public void run() throws InterruptedException {
        while (left > 0L) {
            Thread.sleep(1000);
            left -= 1000;
            onTimeoutUpdate.accept(left);
        }
        onClose.run();
    }
}
```

## 要点总结
- 父组件只管理状态与展示，子组件只负责触发回调上行
- 回调使依赖方向单向，避免子组件直接改父组件状态
- Compose 中通过 state hoisting 保证 UI 更新可预期
